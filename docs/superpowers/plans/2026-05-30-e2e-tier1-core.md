# E2E Tier 1 — Symbol/Analysis Core Specs Implementation Plan (PR-A)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).
> **Repo policy:** commits by the implementing agent within the worktree; final PR + merge via git-agent. Multi-line JSDoc allowed.

**Goal:** Add the 6 Tier-1 user-journey E2E specs (symbol search, analysis render, analysis-job edges + bot-block, 6-tab deep dive, chatbot, model gate) on top of the merged E2E foundation, plus the server-side stubs those journeys need (fake chat LLM + analysis short-circuit).

**Architecture (resolved by code investigation):**
- The **chat** path (`requestChatCompletion`) takes an injectable `callAiProvider: CallAiProvider` — so we inject a **fake chat provider** under `E2E_TEST=1` via a `getLlmProvider()` factory (mirrors `getMarketDataProvider`).
- The **analysis** path (`submitAnalysis` & overall/fundamental/news/options equivalents) does NOT accept an injectable LLM (`SubmitAnalysisOptions` has no `callAiProvider`; core builds the LLM internally and processes via `waitUntil`, no separate worker we can drive in E2E). So we **short-circuit the siglens submit actions under `E2E_TEST=1`** to return `{ status: 'cached', result: <fixture> }` — the client then renders immediately with no poll/worker/LLM. (Realizes the design's §1.3 "network short-circuit" at the action layer.)

**Tech Stack:** Playwright (existing harness), `E2E_TEST` server-side injection, fixtures.

## Pre-flight (resolved design facts — for the implementer)
- LLM chat seam: `src/entities/chat-message/actions/chatAction.ts` calls `requestChatCompletion(opts, { callAiProvider: callAiProviderRouter })`. Router: `src/entities/llm-provider/api/router.ts` → `callAiProviderRouter(options): Promise<string>`. `CallAiProvider` type from `@y0ngha/siglens-core` (`(options: CallAiProviderOptions) => Promise<string>`).
- Analysis submit → `src/entities/analysis/actions/submitAnalysisAction.ts` → core `submitAnalysis(...)`; returns `SubmitAnalysisGatedResult` = `{status:'cached', result} | {status:'submitted', jobId} | {status:'miss_no_trigger'} | {status:'error', error} | ...`. Client hook `src/widgets/symbol-page/hooks/useAnalysis.ts` renders `result` immediately on `'cached'`, polls `pollAnalysisAction(jobId)` on `'submitted'`, shows `BotBlockedNotice` on `'miss_no_trigger'`.
- Overall/fundamental/news/options have parallel submit actions (`submitOverallAnalysisAction`, and the fundamental/news/options analysis actions). Each must get the same `E2E_TEST` short-circuit.
- Model selection: `src/widgets/symbol-page/hooks/useSelectedModel.ts` (localStorage key `LOCAL_STORAGE_ANALYSIS_MODEL_KEY`). Premium gate: `src/features/premium-gate/hooks/useModelGate.ts` (free→allow; premium+no user→`mode:'auth'`; pro tier→allow; non-pro+no BYOK→`mode:'byok'`).
- Bot block: `submitAnalysisAction` sets `skipEnqueueIfMiss = isBot(headers)`; core returns `{status:'miss_no_trigger'}` on cache miss → `setIsBotBlocked(true)` → `BotBlockedNotice` (`src/shared/ui/BotBlockedNotice.tsx`, `role="status"`).
- Reuse selectors from `src/__integration__/`: `analysisFlow`, `newsAnalysisFlow`, `symbolTabNavigation`, `chartTimeframeFlow`, `chartIndicatorFlow`, `modelSelectorFlow`, `optionsPageFlow`, `fearGreedInteraction`, `tickerSearchNavigation`, `recentSearches`. Smoke selectors: `getByRole('banner').getByRole('combobox',{name:'종목 티커 검색'})`, symbol h1 `${displayName} 차트 분석`.
