Free vs Pro: 사용자 구분 전략

구현 난이도와 보안성 사이의 균형을 맞추는 두 가지 접근 방식입니다.

A

로그인 방식 (Full Auth)

NextAuth.js, Clerk 등을 사용한 정석적인 방법입니다.

  • 장점: 확실한 Pro 권한 관리, 분석 히스토리 저장, 기기 간 동기화
  • 단점: 구현 공수 큼, 사용자 가입 허들 발생
B

로그인 없는 방식 (Access Key)

결제 시 이메일로 'Pro 키'를 발급하고, 사이트에서 입력하는 방식입니다.

  • 장점: 가입 절차 없음(저마찰), 구현이 매우 간단함
  • 단점: 키 공유 위험, 브라우저 쿠키/로컬 스토리지 의존

나누는 기준 (Tiering)

항목 Free (로그인X) Pro (인증됨)
AI 모델 Gemini Flash Gemini 2.5 Pro
데이터 일봉 (Daily) 분봉 (1m~60m)
횟수 제한 일 5회 무제한 / 넉넉함

추천 의견

"소소하게" 시작하신다면 B안(Access Key)으로 시작해서 반응을 보신 뒤, 나중에 히스토리 기능이 필요해질 때 A안으로 전환하는 로드맵을 추천합니다.
